## 为什么向量检索还不够？

向量检索擅长理解语义，但有一个明显弱点：

```
用户问：qwen3-rerank 的参数配置怎么写？

向量检索结果：关于"模型参数配置"的通用内容  ← 找到了相关话题
但遗漏了：文档中精确写着 "qwen3-rerank" 的那几行  ← 关键词没命中
```

**原因：** Embedding 对专有名词（模型名、版本号、参数名）不敏感，  
把 "qwen3-rerank" 和 "某个reranker" 的向量差距拉得不够大。

**解决方案：** 同时运行关键词检索，精确匹配专有名词和标题。

## BM25 算法

BM25 是搜索引擎中经典的关键词相关性评分算法（Bing、Elasticsearch 都用它）。  
名字来自"Best Match 25"——这是第 25 次改进版本。

### 核心公式

$$\text{BM25}(q, d) = \sum_{i=1}^{n} \text{IDF}(q_i) \cdot \frac{f(q_i, d) \cdot (k_1 + 1)}{f(q_i, d) + k_1 \cdot (1 - b + b \cdot \frac{|d|}{\text{avgdl}})}$$

看起来复杂，拆开理解其实很直观：

**IDF（逆文档频率）：** 词越稀有，分数越高  
```
"的"出现在所有文档 → IDF 接近 0（没价值）
"qwen3-rerank"只在 2 篇文档出现 → IDF 很高（区分度强）
```

**词频（TF）部分：** 词在文档中出现越多，分数越高，但有上限（饱和）  
```
普通 TF：出现 10 次得分是出现 1 次的 10 倍（不合理）
BM25 TF：出现到一定次数后分数趋于饱和（更合理）
```

**文档长度归一化：**  
长文档天然包含更多词，BM25 按文档长度做归一化，避免长文档虚高。

## BM25 vs 向量检索 对比

| 维度 | BM25 | 向量检索 |
|------|------|---------|
| 专有名词命中 | ✅ 精确匹配 | ⚠️ 较弱 |
| 同义词理解 | ❌ 字面匹配 | ✅ 语义理解 |
| 跨语言检索 | ❌ 不支持 | ✅ 多语言模型支持 |
| 计算成本 | 极低（纯 CPU） | 较高（需要 Embedding API） |
| 离线可用 | ✅ 不需要 API | ❌ 需要 Embedding 服务 |

## 在 LangChain 中使用 BM25

```python
from langchain_community.retrievers import BM25Retriever
from langchain_core.documents import Document

# 构建 BM25 索引（只需要文本，不需要 API）
docs = [
    Document(page_content="qwen3-rerank 是千问的精排模型"),
    Document(page_content="FAISS 是 Meta 开源的向量库"),
    Document(page_content="镜头焦距计算公式 f = WD × sensor / FOV"),
]

bm25_retriever = BM25Retriever.from_documents(docs, k=5)

# 检索
results = bm25_retriever.invoke("qwen3-rerank 怎么用")
for doc in results:
    print(doc.page_content)
```

## 主项目中的实现

`knowledge_base.py` 在构建 FAISS 索引的同时构建 BM25 索引：

```python
def _build_bm25(self, docs):
    if not HAS_BM25 or not docs:
        return
    self.bm25_retriever = BM25Retriever.from_documents(
        docs, k=Config.TOP_K_RESULTS * 2   # 候选数是最终要求的 2 倍，留给 RRF 融合
    )

def _build_bm25_from_store(self):
    # FAISS 加载后，从其内部 docstore 重建 BM25（不需要重新分块）
    all_docs = list(self.vector_store.docstore._dict.values())
    self._build_bm25(all_docs)
```

在 `search()` 方法中，BM25 和 FAISS 并行检索：

```python
faiss_results = retriever.invoke(query)         # 语义检索
bm25_results = self.bm25_retriever.invoke(query) # 关键词检索
print(f"KB 命中：FAISS={len(faiss_results)} BM25={len(bm25_results)}")
```

两路结果通过 RRF 算法融合，详见下一节。
