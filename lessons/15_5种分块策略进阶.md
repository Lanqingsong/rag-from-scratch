# 15 · 5 种分块策略进阶

> 对应主项目 `splitters.py` 全文

---

## 为什么需要多种策略？

第 02 节讲了基础的文本切分，但实际文档的结构千差万别：

| 文档类型 | 最佳策略 |
|---------|---------|
| 有 `##` / `###` 标题的 Markdown | 按标题层级切割，保留完整章节 |
| 编号 Q&A 格式（1.1 / 2.3 ...） | 按题目编号切割，每题一个 chunk |
| 无结构的散文、论文 | 按语义相似度找自然断点 |
| 通用文本 | 递归字符切割（保底方案） |
| 不确定 | 自动探测结构，选最合适策略 |

`splitters.py` 把这 5 种策略封装成统一接口，通过 `kb_config.json` 配置。

---

## 策略一：MarkdownHeadingSplitter

按 Markdown 标题层级切割，每个标题下的内容作为一个 chunk。

```python
class MarkdownHeadingSplitter:
    def __init__(self, heading_level=3, chunk_size=1000, chunk_overlap=0):
        # heading_level=3 → 按 ### 切；2 → 按 ## 切
        prefix = '#' * heading_level
        self._pattern = re.compile(rf'^{prefix}\s+.+', re.MULTILINE)
        deeper = '#' * (heading_level + 1)
        self._sub_pattern = re.compile(rf'^{deeper}\s+.+', re.MULTILINE)
```

**二级切割逻辑：**  
如果某节内容超过 `chunk_size`，自动尝试按下一级标题（`####`）再次切割；  
仍超长则回退到递归字符切割。

```
### 2. 镜头选型              ← 主切割点
#### 2.1 焦距计算            ← 超长时的次级切割点
#### 2.2 分辨率匹配
### 3. 光源选型
```

**适用场景：** 本项目的机器视觉 Markdown 文档，结构清晰，每节知识相对独立。

## 策略二：RegexBoundarySplitter

用正则表达式匹配每个 chunk 的"起始行"，灵活支持各种自定义格式：

```python
class RegexBoundarySplitter:
    def __init__(self, pattern, chunk_size=1000, chunk_overlap=0):
        self.pattern = re.compile(pattern, re.MULTILINE)
```

**使用示例：**

```python
# Q&A 格式："Q1: 什么是..." 
RegexBoundarySplitter(pattern=r'Q\d+[.:：]')

# 编号格式："1.1 镜头焦距..."
RegexBoundarySplitter(pattern=r'\d+\.\d+\s+[^\n]+')

# 中文条款："第一条 ..."
RegexBoundarySplitter(pattern=r'第[一二三四五六七八九十百]+条\s')
```

**工作原理：**  
找到所有匹配位置，相邻两个匹配之间的文本作为一个 chunk。

## 策略三：SemanticSplitter（最智能）

不依赖任何人工规则，通过计算相邻句子之间的 Embedding 相似度来找自然断点：

```python
from langchain_experimental.text_splitter import SemanticChunker

class SemanticSplitter:
    def __init__(self, embeddings, threshold_type='percentile',
                 threshold_value=95, chunk_size=1000):
        self._chunker = SemanticChunker(
            embeddings=embeddings,
            breakpoint_threshold_type=threshold_type,
            breakpoint_threshold_amount=threshold_value,
        )
```

**三种阈值类型：**

| 类型 | 含义 | 推荐值 |
|------|------|-------|
| `percentile` | 相似度下降幅度排名前 X% 处切割 | 95（保守），90（激进） |
| `standard_deviation` | 相似度低于均值 - X 个标准差处切割 | 3 |
| `interquartile` | 低于 Q1 - X×IQR 处切割 | 1.5 |

**工作原理：**

```
句子1 → 向量A
句子2 → 向量B   cos(A,B) = 0.92   ← 相似，同一话题，不切
句子3 → 向量C   cos(B,C) = 0.91
句子4 → 向量D   cos(C,D) = 0.43   ← 相似度骤降，切割！（话题转换）
句子5 → 向量E
```

**缺点：** 每次构建都要调用 Embedding API（有成本），且不适合结构化文档。  
**适用场景：** 无任何结构标记的散文、报告、长文章。

## 策略四：RecursiveSplitter

第 02 节已详细讲过，这里是 `splitters.py` 中的封装版本，作为**最终保底方案**。

```python
def _recursive_split(text, metadata, chunk_size, chunk_overlap):
    from langchain_text_splitters import RecursiveCharacterTextSplitter
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=chunk_size,
        chunk_overlap=chunk_overlap,
        separators=['\n\n', '\n', '。', '！', '？', '；', ' ', ''],
    )
    return splitter.create_documents([text], metadatas=[metadata])
```

注意分隔符列表包含中文标点，这是处理中文文档的关键。

## 策略五：AutoSplitter（自动探测）

不知道文档结构时使用，内置探测算法自动选择最合适的策略：

```python
def _detect_structure(text):
    # 探测优先级：H3 > H2 > H4 > 数字编号（X.X）> 简单编号（X.）> 递归
    if len(_MD_H3.findall(text)) >= 3:
        return 'markdown', {'heading_level': 3}
    if len(_MD_H2.findall(text)) >= 3:
        return 'markdown', {'heading_level': 2}
    if len(_MD_H4.findall(text)) >= 3:
        return 'markdown', {'heading_level': 4}
    if len(_NUMBERED.findall(text)) >= 3:      # 匹配 "1.1 xxx"
        return 'regex', {'pattern': r'\d+\.\d+\s+[^\n]+'}
    if len(_NUMBERED_SIMPLE.findall(text)) >= 3:  # 匹配 "1. xxx"
        return 'regex', {'pattern': r'\d+\.\s+[^\n]+'}
    return 'recursive', {}
```

**关键设计：** 至少 3 个标题才认为是有效结构，避免偶然匹配导致误判。  
如果探测到"无结构"，且传入了 `embeddings`，则启用 SemanticSplitter；  
否则回退到 RecursiveSplitter。

## kb_config.json 配置

每个知识库子目录可以放一个 `kb_config.json`，为该目录独立指定策略：

```json
// 机器视觉文档（Markdown 格式，3 级标题）
{
  "strategy": "markdown",
  "heading_level": 3,
  "chunk_size": 1000,
  "chunk_overlap": 0
}

// Q&A 题库（1.1 格式编号）
{
  "strategy": "regex",
  "pattern": "\\d+\\.\\d+\\s+[^\\n]+",
  "chunk_size": 800
}

// 技术论文（无结构，需语义切割）
{
  "strategy": "semantic",
  "threshold_type": "percentile",
  "threshold_value": 95
}
```

**工厂函数 `create_splitter(config, embeddings)` 根据配置创建对应实例：**

```python
# knowledge_base.py 中的调用
for directory in scan_dirs:
    docs = self._load_raw_documents(directory)
    cfg = self._load_kb_config(directory)        # 读取 kb_config.json，默认 auto
    splitter = create_splitter(cfg, self.embeddings)
    chunks = splitter.split_documents(docs)
```

## 策略选择速查

```
文档有 ##/###/#### 标题？  → markdown
文档是编号 Q&A 格式？      → regex（自定义 pattern）
文档是无结构散文/论文？    → semantic（需要 embeddings）
什么都不确定？             → auto（默认）
快速保底？                 → recursive
```
